# Lab Summary
CybContent is a five-stage AI content pipeline built for IronVeil Security, a cybersecurity company specializing in EDR, ITDR, and MDR for mid-market companies. The core design challenge was producing content that doesn't read as generic AI output — a problem the system addresses through four compounding strategies: hard-coded forbidden phrases in system prompts (no "robust", no "cutting-edge", no "In today's landscape"), a two-tier knowledge base that forces every generation call to be grounded in brand-specific and market-specific context, audience-specific prompt templates that enforce structurally different outputs for CISOs versus SOC analysts, and a human-in-the-loop review step at Stage 4 that feeds directly into a dedicated iteration template.

The two-tier KB architecture was a deliberate choice to solve two separate problems: the primary KB (brand guidelines, product specs, past performance data) handles brand consistency, while the secondary KB (market trends, competitor analysis) handles market positioning. Keeping them separate means either layer can be updated weekly without touching the other. Targeted section-level retrieval rather than full-document injection keeps the pipeline within token limits, which was one of the main technical challenges identified during development.

The pipeline runs through five stages — Document, Monitor, Brief, Publish, Iterate — orchestrated via a CLI in main.py and accessible through a Gradio UI in app.py. The stack is Python, GPT-4o for generation, and Markdown files for the knowledge base, which makes the KB easy to maintain without a database. Output files are timestamped and include generation metadata (model, token count) so content performance can be tracked over time and fed back into the primary KB as historical data, closing the feedback loop.
